MCP Odoo — Setup & Deployment

How to set up, configure, and deploy the MCP Odoo agent

📦 1. Developer Setup

Clone & Install

git clone https://github.com/kathro101/mcp_odoo.git cd mcp_odoo python3 -m venv .venv source .venv/bin/activate pip install -e ".[dev]"

Creates virtual environment with all dependencies (mcp, flask, anthropic, pytest, ruff)

Pre-commit Hooks

pre-commit install

Runs ruff linting, ruff-format, vulture dead-code detection, trailing whitespace cleanup on every commit.

Configure

cp config/config.template.json config/config.json

Edit config.json with Odoo URL, database, username, API key.

🔧 2. Run the Application (3 Modes)

MCP Server (Claude Desktop)

python -m src.mcp_server.server

Runs via stdio — Claude Desktop connects directly. Production mode.

Web UI (Standalone Chat)

python webapp.py

Flask chat at http://localhost:5000. Routes messages through same service layer.

Setup Wizard (DMG Installer)

python installer/wizard.py

Flask wizard at http://localhost:8080. 3-step setup: connect → Claude config → discover schemas.

⚙️ 3. Claude Desktop Configuration

Add to ~/Library/Application Support/Claude/claude_desktop_config.json:

{ "mcpServers": { "odoo": { "command": "python", "args": ["-m", "src.mcp_server.server"], "cwd": "/path/to/mcp_odoo" } } }

After restarting Claude Desktop, 3 tools appear: chat_odoo, list_models, list_agents.

🏗️ 4. Build & Deploy (macOS DMG)

Step 1: Build the .app bundle

bash scripts/build_dmg.sh

PyInstaller packages the Flask wizard + all Python deps into a standalone macOS .app

Step 2: DMG created automatically

dist/MCP-Odoo-Installer.dmg (26 MB)

Ready to distribute. User double-clicks to open the wizard.

Step 3: User runs the wizard

Enters Odoo credentials → tests connection → Claude Desktop auto-configured → schemas discovered
