MCP Odoo — System Architecture

How Claude Desktop talks to Odoo ERP — the complete data flow

1
Claude Desktop — THE AI BRAIN

User sends a message

User: "Create a shipment for ACME Corp, road direct, booking ref BOOKREF123"

Claude understands the intent and calls the chat_odoo MCP tool.

↓ JSON-RPC over stdio
2
MCP Server (src/mcp_server/)

Route & Enrich

Keyword routing: "shipment" → matches Logistics agent → stock.picking model

Schema lookup: loads config/schemas/stock_picking.json — 25 fields, aliases, workflow hints

Returns: enriched schema with FIELD ALIASES, WORKFLOW HINTS, REQUIRED FIELDS, selection options, help text

↑ Structured schema response
3
Claude Desktop — THE AI BRAIN

Map user words → field names using aliases

"ACME Corp" → partner_id (via "customer" alias)

"road direct" → picking_type_id (via "type" alias)

"BOOKREF123" → origin (via "booking"/"ref" aliases, guided by help text)

Workflow hint: dates apply to milestone lines, not header

↓ chat_odoo(action="preview", model="stock_picking", params={...})
4
MCP Server (src/mcp_server/)

Preview & Validate

Calls src/operations/create.py::preview_record()

Checks required fields: ✅ partner_id, ❌ name, ❌ picking_type_id

Returns: {status: "needs_input", missing: ["name", "picking_type_id"], field_aliases: {...}}

↑ Preview result
5
Claude Desktop — THE AI BRAIN

Fill missing fields, confirm with user

Claude asks: "Should I name this SHIP/001 and use Delivery Orders?"

User confirms → Claude sends action="update" with full params

↓ chat_odoo(action="update", model="stock_picking", params={...}, record_id=42)
6
MCP Server → Operations Layer

Execute Against Odoo

Calls src/operations/update.py::update_record()

Delegates to src/odoo_service/odoo_client.py::execute_kw()

XML-RPC: stock.picking.write([42], {name: "SHIP/001", ...})

↓ XML-RPC
7
Odoo ERP

Process & Respond

Odoo validates, writes the record, returns True

↑ XML-RPC response
8
Claude Desktop

Confirm to user

Claude: "✅ Shipment SHIP/001 created for ACME Corp. Booking ref: BOOKREF123."

📊 Actual Data Exchange (shipment creation)

Claude → MCP Server:
  chat_odoo(message="Create a shipment for ACME Corp, road direct, BOOKREF123")

MCP Server → Claude:
  ## Routed to: Logistics Agent
  ## Model: Transfers (stock.picking)

  ### FIELD ALIASES
    "customer" → partner_id
    "type" → picking_type_id
    "booking" → origin

  ### REQUIRED FIELDS
    - partner_id (many2one → res.partner): Contact *REQUIRED*
    - picking_type_id (many2one → stock.picking.type): Operation Type *REQUIRED*

Claude → MCP Server:
  chat_odoo(action="search", model="res.partner", params={name: "ACME Corp"})

MCP Server → Claude:
  {status: "success", records: [{id: 42, name: "ACME Corp"}]}

Claude → MCP Server:
  chat_odoo(action="preview", model="stock_picking",
    params={partner_id: 42, origin: "BOOKREF123"})

MCP Server → Claude:
  {status: "needs_input", filled: [partner_id, origin],
   missing: [name, picking_type_id], field_aliases: {...}}

... Claude asks user for missing fields ...

Claude → MCP Server:
  chat_odoo(action="update", model="stock_picking", record_id=42,
    params={name: "SHIP/001", partner_id: 42, picking_type_id: 3, origin: "BOOKREF123"})

MCP Server → Odoo:
  XML-RPC: stock.picking.write([42], {name: "SHIP/001", ...})

Odoo → MCP Server → Claude:
  {status: "success", record_id: 42}

Claude → User:
  ✅ Shipment SHIP/001 created for ACME Corp. Booking ref: BOOKREF123.
